# AI 导演成片工作台页面规格

## 目标用户

这个网页给内测创作者、导演型用户和短视频团队使用。用户不想只看到一个黑盒生成按钮，而是希望从一个想法开始，逐步看到剧本、角色、场景、道具、分镜、素材、剪辑和最终成片，并且每一步都能用自然语言继续调整。

## 产品目标

把“一句话生成视频”改造成“可验收的 AI 导演流程”：

1. 输入想法后，先生成剧本和可视化步骤，而不是直接跳到最终成片。
2. 每个步骤都要显示当前状态、生成结果、可编辑内容和下一步动作。
3. 用户能对剧本、角色、场景、道具、分镜、剪辑分别下自然语言指令。
4. 用户能看到 LibTV 返回的图片、视频、画布链接和最终成片。
5. 用户能选用某个素材版本，让后续成片优先使用它。

## 第一屏必须看到

桌面端第一屏必须同时看到：

- 左侧输入区：项目想法、风格、片长、生成剧本和可视化步骤按钮。
- 左侧最近任务：最近 5-8 个任务和状态。
- 右侧当前任务状态：如果还没任务，显示“输入想法后会生成什么”。
- 有任务后右侧顶部必须显示“当前结果总览”，包括剧本、角色、场景、道具、分镜、成片六个验收块。
- 每个验收块必须有“查看”入口，可以跳到对应区域。

手机端第一屏必须看到：

- 项目想法输入区。
- 生成按钮。
- 当前结果总览排在输入区下面。
- 最近任务可以下移，不抢第一屏。

## 页面区域

### 1. 顶部状态栏

内容：

- 产品名：一个想法立刻成片。
- Provider 状态：GPT-5.4、LibTV、FFmpeg。
- 不把缺失的备用 provider 放在强视觉位置，避免用户误以为主链不可用。

验收：

- GPT-5.4 ready 和 LibTV ready 一眼可见。
- 本地 Remotion 缺失不能看起来像主流程失败。
- 移动端不溢出、不横向滚动。

### 2. 输入区

内容：

- 项目想法 textarea。
- 风格选择。
- 片长选择。
- 主按钮：生成剧本和可视化步骤。
- 一句说明：先拆步骤，确认后再生成素材和成片。

验收：

- 输入一句话后必须先返回剧本和步骤。
- 主按钮 loading 文案明确，例如“剧本生成中...”。
- 不允许用户以为点一次就已经完成全部素材生成。

### 3. 当前结果总览

内容：

- 剧本：标题、梗概、状态。
- 角色/演员：卡片数量、素材完成数、查看入口。
- 场景：卡片数量、素材完成数、查看入口。
- 道具：卡片数量、素材完成数、查看入口。
- 分镜：文字分镜数量、视频素材完成数、查看入口。
- 成片：最终视频状态、查看入口。

验收：

- 输入想法后，这个区域必须出现在右侧顶部。
- 每张验收卡都能说明“现在有什么”和“还缺什么”。
- 点击查看能跳到对应模块。
- 没有素材时也要显示草稿名称，不能空白。

### 4. 导演指令区

内容：

- 一个全局自然语言输入框。
- 三个动作：只调整步骤、调整并生成素材、串成成片。
- 显示最近一次 GPT 调度回复。

验收：

- 用户能用一句话改任意模块。
- “只调整步骤”不消耗 LibTV。
- “调整并生成素材”会提交步骤素材。
- “串成成片”会把当前选用素材和上下文交给 LibTV。

### 5. 剧本区

内容：

- 标题。
- 故事梗概。
- 整片旁白。
- 文字分镜卡片：镜头编号、标题、画面说明、旁白。

验收：

- 输入想法后必须能看到剧本。
- 剧本必须比原始想法更结构化。
- 分镜至少 3 条或按目标片长拆分。
- 剧本区有稳定锚点，能从总览跳转。

### 6. 可调步骤区

分组：

- 角色/演员。
- 场景。
- 道具。
- 分镜。

每组内容：

- 自然语言调整输入框。
- 调整本组按钮。
- 调整并生成本组素材按钮。
- 直接生成本组素材按钮。
- 多张可编辑卡片。

每张卡片内容：

- 名称。
- 设定。
- 交给 LibTV 的任务。
- 如果是分镜，还要有时长、镜头、旁白。
- 保存修改。
- 用 LibTV 生成。
- 同步。
- 画布链接。
- 返回素材列表。
- 选用按钮。

验收：

- 每个卡片必须能编辑并保存。
- 每个卡片必须能单独交给 LibTV。
- LibTV 有结果后必须显示图片或视频。
- 多个结果必须能选用一个。
- 选用结果必须进入剪辑面板和最终成片上下文。

### 7. 剪辑面板

内容：

- 素材池。
- 预览窗口。
- 检查器。
- 时间线 V1/A1。
- 剪辑自然语言指令。
- 用当前时间线成片按钮。

验收：

- 素材池优先显示已选用素材。
- 时间线按分镜顺序显示。
- 没有素材时显示明确占位。
- 有最终视频时预览窗口直接播放最终视频。
- 移动端变成单列，不重叠。

### 8. 最终成片区

内容：

- 视频播放器。
- 同步 LibTV 成片按钮。
- 直接打开视频链接。
- 打开 LibTV 成片画布链接。
- 错误信息。

验收：

- LibTV 返回视频后能播放。
- LibTV 超时但有画布链接时，用户能稍后同步。
- 错误信息必须面向用户，不暴露 key。

## 风格参考

参考方向：

- 达芬奇剪辑面板：素材池、预览、检查器、时间线的工作台布局。
- TapNow/LTX 类 storyboard：从剧本到镜头卡片，每一步能验收。
- Runway/Vidu 类素材版本：参考图、视频版本、选用状态明确。

视觉要求：

- 内测工具，不做营销官网。
- 信息密度高，但层级清楚。
- 卡片圆角不超过 8px。
- 使用克制的白底、灰线、深色时间线和少量强调色。
- 不用大面积渐变、装饰球、营销 hero。

## 明确不要

- 不要做落地页。
- 不要只有一个输入框和一个最终视频。
- 不要把关键结果藏到页面很下面且没有入口。
- 不要让“备用 provider 缺失”看起来像主流程失败。
- 不要让按钮文案暗示一步完成全部流程。
- 不要在没有素材时留大片空白。

## 响应式规则

桌面端：

- 左侧固定输入栏，右侧为结果工作台。
- 右侧顶部固定顺序：状态、全流程、当前结果总览、导演指令。
- 剧本、步骤、剪辑、成片依次向下。

手机端：

- 单列布局。
- 输入区在最上方。
- 当前结果总览紧跟输入区。
- 最近任务下移。
- 可调步骤卡片单列。
- 剪辑面板单列，时间线可横向滚动。

## 任务包拆分

### 任务包 1：首屏和结果总览

范围：

- 输入区。
- 当前状态。
- 全流程。
- 当前结果总览。
- 查看跳转。

验收：

- 输入想法后，右侧顶部能看到剧本、角色、场景、道具、分镜、成片六块状态。
- 每块都能说明当前结果。
- 每块都有查看入口。
- 1440px 和 390px 不重叠、不溢出。

### 任务包 2：剧本可视化

范围：

- 剧本摘要。
- 旁白。
- 文字分镜卡片。
- 剧本自然语言调整。

验收：

- GPT 返回后立刻显示标题、梗概、旁白和分镜。
- 每条分镜有镜头标题、画面说明、旁白。
- 用户能用自然语言调整剧本，不触发 LibTV。

### 任务包 3：角色/场景/道具主体图

范围：

- 三组素材卡。
- 单卡保存。
- 单卡生成。
- 本组生成。
- 素材展示和选用。

验收：

- 每组至少有一张可编辑卡。
- LibTV 返回图后能显示。
- 每张图有选用状态。
- 选用图进入素材池。

### 任务包 4：分镜到视频镜头

范围：

- 分镜卡片。
- 图片分镜。
- 视频镜头。
- 按顺序生成。

验收：

- 分镜卡显示时长、镜头、旁白。
- 每条分镜能生成素材。
- 返回视频后能在卡片里播放。
- 时间线按分镜顺序显示完成状态。

### 任务包 5：剪辑面板

范围：

- 素材池。
- 预览窗口。
- 检查器。
- V1/A1 时间线。
- 剪辑指令。

验收：

- 素材池显示所有已返回素材。
- 选用素材排在前面。
- 时间线不随内容高度乱跳。
- 移动端单列可用。

### 任务包 6：最终成片和同步

范围：

- 串成成片。
- LibTV 画布链接。
- 轮询和手动同步。
- 视频播放。

验收：

- 提交成片后能看到等待状态。
- 返回视频后能播放。
- 超时后保留同步按钮。
- 错误信息清楚可理解。

### 任务包 7：截图 QA 和视觉修复

范围：

- 1440px 桌面。
- 390px 手机。
- 长文本。
- 无素材、生成中、有素材、已成片状态。

验收：

- 无横向滚动。
- 按钮文字不溢出。
- 卡片不重叠。
- 视频和图片不把布局撑坏。
- 第一屏能解释“现在发生了什么”。

## 下一步建议

当前最应该做任务包 1 和任务包 2 的细化验收：先把“输入想法后马上看到剧本和步骤”做到稳定，再继续补主体图和分镜视频自动串联。
