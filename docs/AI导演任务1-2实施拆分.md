# AI 导演任务 1-2 实施拆分

更新时间：2026-04-28

## 1. 目标

把当前优先级最高的两个任务包拆成可直接交给 Agent 执行的小任务：

- 任务 1：首屏和结果总览。
- 任务 2：剧本可视化。

拆分原则：

- 每个小任务都要有明确范围。
- 每个小任务都要有可见输出。
- 每个小任务都要有验收标准。
- 不安装新工具，不引入新框架。
- 不大规模改页面结构。

## 2. 当前已具备

当前网页已有：

- 左侧输入区。
- 最近任务。
- 顶部 provider 状态。
- 当前状态卡。
- 全流程步骤。
- 当前结果总览。
- 导演指令。
- 剧本区。
- 可调步骤区。
- 剪辑面板。
- 最终成片区。

当前文档已有：

- `docs/AI导演工作台页面规格.md`
- `docs/AI导演网页规范化推进计划.md`
- `docs/AI导演网页Agent执行规范.md`
- `docs/AI导演网页验收矩阵.md`
- `docs/AI导演竞品观察-小云雀.md`

## 3. 任务 1：首屏和结果总览

### 任务 1.1：空状态首屏说明

范围：

- `public/app.js` 的空任务结果区。
- 必要时少量调整 `public/styles.css`。

输出：

- 没有任何任务时，右侧不要空白。
- 显示“输入想法后会生成什么”。
- 展示 4 个步骤预告：剧本、主体图、分镜视频、剪辑成片。

验收：

- 还没有任务时，用户能理解下一步是输入想法。
- 不显示误导性的错误状态。
- 1440px 首屏能看到输入区和空状态说明。
- 390px 输入区后能看到空状态说明。

不做：

- 不调用 GPT。
- 不调用 LibTV。
- 不改历史任务逻辑。

### 任务 1.2：Provider 状态降噪

范围：

- 顶部 provider 展示逻辑。

输出：

- 第一优先显示 GPT-5.4、LibTV、FFmpeg。
- 备用 provider 缺失可以折叠或弱化。
- Remotion 缺失不能看起来像主链失败。

验收：

- 用户第一眼知道主链是否可用。
- 页面不因一堆 missing/error 让人误判系统坏了。
- 移动端 provider 不横向撑破。

不做：

- 不删除 provider 状态接口。
- 不改 provider 配置。

### 任务 1.3：结果总览状态升级

范围：

- 当前结果总览卡片。

输出：

- 每张卡显示：
  - 当前结果。
  - 还缺什么。
  - 下一步动作。
- 角色、场景、道具、分镜显示“待生成素材/已有素材/待选用/已选用”。
- 成片显示“待提交/生成中/可同步/已完成”。

验收：

- 有剧本无素材时，不会误以为流程失败。
- 有素材未选用时，提示先选素材。
- 已成片时，提示可播放或继续调整。
- 每张卡都有查看入口。

不做：

- 不重新设计所有卡片。
- 不新增复杂状态数据库。

### 任务 1.4：总览到模块的跳转 QA

范围：

- 总览卡 `查看` 链接。
- 各模块锚点。

输出：

- 剧本跳到剧本区。
- 角色跳到角色区。
- 场景跳到场景区。
- 道具跳到道具区。
- 分镜跳到分镜区。
- 成片跳到最终成片区。

验收：

- 每个链接可点击。
- 跳转后标题不被顶部遮挡。
- 链接文案统一。

不做：

- 不做复杂滚动动画。

### 任务 1.5：首屏截图 QA

范围：

- 当前页面。

输出：

- 1440 x 900 人工 QA 记录。
- 390 x 844 人工 QA 记录。

验收：

- 无横向滚动。
- 结果总览不被折到太下面。
- 按钮不溢出。
- provider 不挤爆顶部。

不做：

- 不安装 Playwright。

## 4. 任务 2：剧本可视化

### 任务 2.1：剧本区层级整理

范围：

- 剧本区展示。

输出：

- 剧本标题。
- 故事梗概。
- 整片旁白。
- 文字分镜。
- 视觉层级更清楚：先总后分镜。

验收：

- GPT 返回后用户能一眼看到故事是什么。
- 旁白不会和梗概混在一起。
- 分镜卡片不撑破。

不做：

- 不改变 GPT 输出结构。

### 任务 2.2：导演公式卡

来源：

- 小云雀“AI 思考过程/创意公式”机制。

范围：

- 剧本区新增一个“导演公式”卡。
- 初期可从现有 story/workspace 推导展示，不强制后端新增字段。

输出字段：

- 主题。
- 情绪。
- 冲突。
- 主视觉。
- 角色。
- 场景。
- 道具。
- 节奏。

验收：

- 输入想法后，用户能看到 AI 如何拆解故事。
- 没有某个字段时显示“待补充”，不空白。
- 公式卡不触发 LibTV。
- 用户知道可用导演指令修改公式和剧本。

不做：

- 不新增复杂 schema。
- 不改变 OpenRouter 提示词，除非后续任务需要。

### 任务 2.3：剧本专用自然语言调整入口

范围：

- 剧本区内增加一个轻量调整入口，或把全局导演指令链接到剧本调整。

输出：

- 文案示例：`例如：把结尾改成反转，旁白更克制，分镜增加雨夜便利店的反光。`
- 按钮：`只调整剧本`

验收：

- 用户不用回到页面顶部也知道剧本可以改。
- 只调整剧本不触发 LibTV。
- 调整后剧本区更新。

不做：

- 不新增多套重复命令系统；如果复用现有 director-command，要保持接口一致。

### 任务 2.4：文字分镜验收状态

范围：

- 文字分镜卡片。

输出：

- 每条分镜显示：
  - 镜头编号。
  - 标题。
  - 画面说明。
  - 旁白。
  - 对应素材状态：待生成素材/已生成/已选用。

验收：

- 用户能理解“文字分镜”和“视频镜头”不是同一步。
- 未生成视频时不显示失败感。
- 已生成视频时能引导去分镜卡或剪辑面板查看。

不做：

- 不直接自动生成视频。

### 任务 2.5：剧本长文本 QA

范围：

- 剧本区和文字分镜卡片。

输出：

- 长标题、长梗概、长旁白、长分镜说明的人工 QA 记录。

验收：

- 卡片不重叠。
- 文本不溢出按钮或容器。
- 手机端可读。

不做：

- 不安装自动化测试工具。

## 5. 推荐执行顺序

第一轮：

1. 任务 1.1：空状态首屏说明。
2. 任务 1.3：结果总览状态升级。
3. 任务 2.2：导演公式卡。

第二轮：

1. 任务 1.2：Provider 状态降噪。
2. 任务 2.1：剧本区层级整理。
3. 任务 2.4：文字分镜验收状态。

第三轮：

1. 任务 1.4：总览跳转 QA。
2. 任务 1.5：首屏截图 QA。
3. 任务 2.5：剧本长文本 QA。

## 7. 下一步

下一次实际页面实现建议从任务 1.1 开始：先把无任务空状态做好。它风险最低，但能显著改善用户第一眼“不知道结果在哪里”的问题。
