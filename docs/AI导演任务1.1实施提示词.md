# AI 导演任务 1.1 实施提示词：空状态首屏说明

更新时间：2026-04-28

## 1. 任务名称

任务 1.1：空状态首屏说明

## 2. 任务目标

当用户还没有创建任何任务时，右侧结果区不能只显示“等待创建任务 / 先提交一个想法”。它必须像一个工作台入口，明确告诉用户：

- 输入想法后会先得到剧本。
- 然后会得到角色/场景/道具主体图。
- 然后会得到分镜视频。
- 最后会进入剪辑和成片。

这个任务只改善无任务空状态，不调用 GPT，不调用 LibTV，不改历史任务逻辑。

## 3. 开发前必须阅读

先读：

1. `docs/AI导演工作台页面规格.md`
2. `docs/AI导演网页规范化推进计划.md`
3. `docs/AI导演网页Agent执行规范.md`
4. `docs/AI导演网页验收矩阵.md`
5. `docs/AI导演任务1-2实施拆分.md`

## 4. 只允许改动

允许：

- `public/app.js`
- `public/styles.css`

不允许：

- 不改后端接口。
- 不改 GPT 提示词。
- 不改 LibTV 调用。
- 不改任务存储。
- 不安装新依赖。

## 5. 必须输出

无任务时右侧结果区应包含：

1. 状态卡：`等待创建任务`
2. 一句清楚说明：`输入一个想法后，会先生成剧本和可视化步骤，再生成素材和成片。`
3. 四个步骤预告卡：
   - 剧本拆解：标题、梗概、旁白、文字分镜。
   - 主体图：角色、场景、道具。
   - 分镜视频：按镜头生成图片分镜和视频镜头。
   - 剪辑成片：素材池、时间线、配音配乐、最终视频。
4. 一个轻量提示：`第一步先在左侧输入项目想法。`

## 6. 建议 UI 结构

在 `renderResult()` 的 `if (!job)` 分支里，把当前简单状态卡替换为更完整的空状态：

```html
<section class="status-card">
  ...
</section>
<section class="panel-block empty-workbench">
  <div class="section-head wrap">...</div>
  <div class="empty-flow-grid">
    <article>剧本拆解...</article>
    <article>主体图...</article>
    <article>分镜视频...</article>
    <article>剪辑成片...</article>
  </div>
</section>
```

样式放在 `public/styles.css`：

- `.empty-workbench`
- `.empty-flow-grid`
- `.empty-flow-card`

## 7. 验收标准

桌面 1440 x 900：

- 左侧输入区和右侧空状态能同屏看到。
- 右侧不是空白后台。
- 四个步骤卡不重叠。
- 按钮和文字不溢出。

手机 390 x 844：

- 输入区在上方。
- 空状态紧跟输入区。
- 四个步骤卡变成单列或自然换行。
- 无横向滚动。

功能：

- 不创建任务时不调用 `/api/generate`。
- 不调用 LibTV。
- 不改变最近任务逻辑。
- 页面刷新无报错。

## 9. 禁止事项

- 不要把空状态做成营销 landing page。
- 不要加入大图、渐变 hero、装饰背景。
- 不要新增“开始生成全部成片”这类误导按钮。
- 不要让用户以为第一步会直接生成最终视频。
- 不要隐藏左侧输入区。
