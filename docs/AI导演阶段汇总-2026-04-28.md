## 3. GitHub/开源方法学到什么

### 3.1 awesome-cursorrules

学到：

- Agent 需要项目级规则。
- 规则要写清技术栈、结构、禁止事项、测试标准。
- 临时口头描述不稳定，要沉淀成文档。

本项目落地：

- 已新增 `AI导演网页Agent执行规范.md`。
- 要求后续 Agent 先读规格再改页面。

### 3.2 Playwright

学到：

- 网页验收要用真实浏览器、多视口、真实交互。
- 截图 QA 能发现代码阅读发现不了的问题。

本项目落地：

- 已新增 `AI导演网页验收矩阵.md`。
- 暂不安装 Playwright，先用 in-app browser 做人工 QA。

### 3.3 Storybook

学到：

- UI 不能只看一个完成状态。
- 每个模块都要有 empty/loading/error/completed 状态。

本项目落地：

- 在验收矩阵里固定了无任务、生成中、有剧本无素材、有素材、已成片、失败/超时、长文本状态。

### 3.4 axe-core

学到：

- 可访问性不是最后加的，要在按钮、表单、状态文案里体现。

本项目落地：

- 要求按钮文案清晰，输入框有 label，状态不能只靠颜色表达。

### 3.5 Lighthouse

学到：

- 性能和可用性会被重素材拖垮。
- 视频/图片不能无限加载。

本项目落地：

- 验收矩阵要求大量 LibTV 素材折叠展示，只加载代表结果。

## 4. 竞品学到什么

### 小云雀

最重要机制：

- 能力入口分类。
- AI 思考过程/创意公式。
- 一次生成多个候选，用户选一个。
- 自有素材和 AI 素材融合。
- 排队时间要解释清楚。

本项目最该吸收：

- 新增“导演公式卡”：主题、情绪、冲突、主视觉、角色、场景、道具、节奏。
- 总览层显示“已选用/待选用”。
- LibTV 生成等待文案更像排队生成，而不是失败。

### TapNow / LTX 类 storyboard

最重要机制：

- 剧本自动拆成可编辑 storyboard。
- 镜头卡有状态，不只是文本。
- 用户可以逐镜头验收。

本项目最该吸收：

- 文字分镜卡显示对应视频状态。
- 分镜和视频镜头清楚区分。

### Runway / Vidu 类参考素材

最重要机制：

- 参考图、参考主体、候选版本会影响后续生成。
- 多候选必须有选用状态。

本项目最该吸收：

- 选用状态回到总览。
- 后续再补参考素材上传入口。

### 达芬奇式剪辑面板

最重要机制：

- 素材池、预览、检查器、时间线要在一个工作台里表达清楚。

本项目最该吸收：

- 剪辑面板保留现有结构，但需要截图 QA 和移动端打磨。

## 5. 当前网页最大问题

### P0：无任务空状态太弱

用户第一次打开页面时，不知道输入想法后会生成什么。

下一步：

- 做任务 1.1：空状态首屏说明。

### P0：Provider 状态容易误导

备用能力 error 会让用户误判主链坏了。

下一步：

- 做任务 1.2：Provider 状态降噪。

### P0：缺少导演公式中间层

用户看不到 AI 怎么把想法拆成可控结构。

下一步：

- 做任务 2.2：导演公式卡。

### P1：结果总览没有充分表达“还缺什么”

用户不知道下一步应该生成素材、选素材，还是串成片。

下一步：

- 做任务 1.3：结果总览状态升级。

### P1：文字分镜和视频镜头关系不够清楚

用户可能误以为有文字分镜就已经生成视频。

下一步：

- 做任务 2.4：文字分镜验收状态。

## 7. 首批 5 个可执行小任务

1. 任务 1.1：空状态首屏说明。
2. 任务 1.2：Provider 状态降噪。
3. 任务 1.3：结果总览状态升级。
4. 任务 2.2：导演公式卡。
5. 任务 2.4：文字分镜验收状态。

推荐顺序：

1. 先做 1.1，因为风险最低、用户感知最大。
2. 再做 1.2，避免主链 ready 时被备用 error 干扰。
3. 再做 1.3，让用户知道还缺什么。
4. 再做 2.2，把小云雀的创意公式机制并入剧本区。
5. 再做 2.4，让分镜和视频镜头关系清楚。

## 8. 暂不安装的工具

暂不安装：

- Playwright。
- Storybook。
- axe-core。
- Lighthouse。

原因：

- 当前项目还在快速原型阶段。
- 用户选择今晚先要完整规划。
- 安装第三方工具需要单独确认。

替代方案：

- 用 in-app browser 做人工截图 QA。
- 用文档矩阵做检查清单。
- 后续确定进入稳定开发阶段后，再考虑自动化工具。
