## 2. 今晚要交付什么

### 2.1 GitHub/开源工具学习摘要

必须覆盖：

- Agent 规则库：awesome-cursorrules。
- 浏览器验收：Playwright。
- 组件状态隔离：Storybook。
- 可访问性检查：axe-core。
- Web 质量审计：Lighthouse。

每个工具输出：

- 它解决什么问题。
- 当前项目是否立即采用。
- 如果暂不采用，用什么手工流程替代。
- 未来安装前需要用户确认什么。

### 2.2 竞品机制学习摘要

必须覆盖：

- 小云雀。
- TapNow/LTX 类 storyboard。
- Vidu/Runway 类参考素材和多候选。
- 达芬奇式剪辑面板。

每个竞品或机制输出：

- 可借鉴机制。
- 当前网页已有部分。
- 当前网页差距。
- 对应任务包。

### 2.3 当前网页问题诊断

按用户体验列问题，不按代码文件列问题：

- 输入想法后，用户是否知道结果在哪里。
- 剧本是否足够可视化。
- 角色/场景/道具是否能验收。
- 分镜和视频镜头是否被清楚区分。
- 素材选用是否影响后续成片。
- LibTV 等待/超时是否可理解。
- 手机端是否能用。

每个问题要标：

- 严重程度：高/中/低。
- 影响用户。
- 对应任务包。
- 建议下一步。

### 2.4 任务包总表

必须包含：

- 任务 0：Agent 执行规范。
- 任务 1：首屏和结果总览。
- 任务 2：剧本可视化。
- 任务 3：角色/场景/道具主体图。
- 任务 4：分镜到视频镜头。
- 任务 5：剪辑面板。
- 任务 6：最终成片和同步。
- 任务 7：截图 QA 和视觉修复。

每个任务必须包含：

- 目标。
- 输出。
- 验收标准。
- 当前状态。
- 下一步小任务。

### 2.5 首批可执行任务清单

优先级建议：

1. 任务 1.1：空状态首屏说明。
2. 任务 1.3：结果总览状态升级。
3. 任务 2.2：导演公式卡。
4. 任务 1.2：Provider 状态降噪。
5. 任务 2.4：文字分镜验收状态。

每个任务输出：

- 只允许改哪里。
- 不允许改哪里。
- 验收标准。
- 完成后如何汇报。

### 2.6 截图 QA 计划

必须包含：

- 1440 x 900 桌面。
- 390 x 844 手机。
- 无任务。
- 有剧本无素材。
- 有素材未成片。
- 已成片。
- 失败/超时。
- 长文本。

检查项：

- 第一屏是否解释清楚。
- 结果是否有入口。
- 按钮是否溢出。
- 卡片是否重叠。
- provider 是否误导。
- 视频/图片是否撑坏页面。
