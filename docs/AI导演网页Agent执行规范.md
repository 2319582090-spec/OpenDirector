# AI 导演网页 Agent 执行规范

更新时间：2026-04-28

## 1. 开始前必须阅读

每个网页开发 Agent 开始前必须先读：

1. `docs/AI导演工作台页面规格.md`
2. `docs/AI导演网页规范化推进计划.md`
3. 当前任务对应的任务包说明

不要在未读规格的情况下直接改页面。

## 2. 工作原则

- 先理解当前页面结构，再决定改哪里。
- 只改当前任务包范围内的内容。
- 不做落地页，不做营销页。
- 不引入新框架，除非用户明确同意。
- 不自动安装第三方插件或运行新下载脚本。
- 不暴露 API key、访问令牌或内部堆栈。
- 用户看到的文案必须偏产品语言，少用技术术语。
- 每个页面状态都要回答用户三个问题：现在有什么、还缺什么、下一步点哪里。

## 3. 改动前说明

动手前必须简短说明：

- 当前任务包名称。
- 准备改哪些区域。
- 不会改哪些无关区域。
- 验收标准是什么。

## 4. UI 验收底线

任何 UI 任务都必须检查：

- 桌面端 1440px。
- 手机端 390px。
- 无任务空状态。
- 有剧本无素材状态。
- 有素材状态。
- 已成片状态。
- 长文本状态。

必须避免：

- 横向滚动。
- 按钮文字溢出。
- 卡片重叠。
- 视频或图片撑坏布局。
- provider 错误误导用户认为主链失败。
- 用户点完按钮后不知道结果在哪里。

## 5. 交互验收底线

核心交互必须可用：

- 输入想法后生成剧本和可视化步骤。
- 当前结果总览能跳到对应模块。
- 剧本、角色、场景、道具、分镜都能被自然语言调整。
- 单张卡片能保存。
- 单张卡片能交给 LibTV。
- LibTV 结果能同步和展示。
- 素材能选用。
- 最终成片能提交、同步和播放。

如果某个交互暂时不能实现，必须在页面上给用户明确解释，不留空白。

## 6. 每个任务包的完成定义

一个任务包只有同时满足以下条件才算完成：

- 目标区域出现在页面上。
- 空状态、生成中、已生成状态都有合理展示。
- 至少桌面和手机各检查一次。
- 没有明显文字溢出和布局重叠。
- 关键按钮能点击并有反馈。
- 完成汇报包含改动、验证、剩余问题。

## 7. 完成汇报格式

每次完成一个任务包后，用这个格式汇报：

```text
完成：[任务包名称]

改动：
- [改动 1]
- [改动 2]

验证：
- [检查项 1]
- [检查项 2]

剩余问题：
- [问题或“暂无明显阻塞”]

下一步：
- [建议的下一个小任务]
```

## 8. 禁止事项

- 不要把大任务一次性全做完。
- 不要只改样式不检查截图。
- 不要把关键结果藏在页面很下面且没有入口。
- 不要让用户以为“生成剧本”已经等于“生成成片”。
- 不要在已有结构可用时另起一套 UI 系统。
- 不要在没有用户确认时安装新工具。

## 9. 推荐小步顺序

1. 先修首屏和结果总览。
2. 再修剧本可视化。
3. 再修角色/场景/道具主体图。
4. 再修分镜到视频镜头。
5. 再修剪辑面板。
6. 再修最终成片和同步。
7. 最后做截图 QA 和视觉打磨。
