# 项目拆分总表

更新时间：2026-04-22

## 1. 为什么现在拆分

当前项目已经不只是“做一个能调 API 的网页”，而是在做一个有工作台、有模型编排、有导演调度能力的 Web 产品。

如果继续把所有事情混在一起推进，会出现 3 个问题：

- `UI` 会和真实能力脱节，容易做成假壳子
- `API` 会越接越乱，没有统一测试标准
- `导演 Agent` 会被界面和 provider 细节拖慢

所以现在固定拆成 3 条主线并行推进。

## 2. 三条主线

### A. 网页 UI 主线

目标：

- 把工作台做成可看、可用、可理解的产品界面

负责范围：

- 页面结构
- 底部 6 步流程
- 三栏工作区
- 空状态 / 错误状态 / 任务状态展示
- 项目结果预览

当前对应文件：

- [index.html](../public/index.html)
- [styles.css](../public/styles.css)
- [app.js](../public/app.js)

### B. API 与模型接入主线

目标：

- 接稳文本、图片、视频、配音能力，并保持可替换、可测试、可 fallback

负责范围：

- provider 接入
- key 配置
- 调用协议适配
- 错误处理
- provider 状态可视化

当前对应文件：

- [config.js](../src/server/config.js)
- [provider-status.js](../src/server/provider-status.js)
- [script.js](../src/server/providers/script.js)
- [dashscope-video.js](../src/server/providers/dashscope-video.js)
- [runway-video.js](../src/server/providers/runway-video.js)
- [voice.js](../src/server/providers/voice.js)

### C. 导演 Agent 与产品开发主线

目标：

- 让系统从“单次调用工具”升级成“会做总调度的导演 Agent”

负责范围：

- idea -> plan -> story 的编排
- 上下文记忆
- provider 选择
- 失败 fallback
- 任务分阶段执行
- job 数据结构设计

当前对应文件：

- [pipeline.js](../src/server/pipeline.js)
- [job-store.js](../src/server/job-store.js)
- [story.js](../src/server/utils/story.js)

## 3. 三条主线的关系

### UI 依赖 API

- 没有真实 provider 状态，UI 只能展示假数据

### API 依赖导演 Agent

- 没有调度逻辑，provider 只是“接口堆积”，不能形成产品

### 导演 Agent 依赖 UI

- 没有清晰工作台，用户看不懂系统在做什么

所以正确顺序不是“先做完其中一个”，而是：

1. 每条线都先做最小闭环
2. 每条线都给出清晰产物
3. 再逐步把三条线粘合起来

## 4. 当前优先级

### 第一优先级

- `导演 Agent / 产品开发`

原因：

- 这是整个产品的核心差异，不解决这个问题，网站再漂亮也只是“API 壳”

### 第二优先级

- `API / 模型接入`

原因：

- 调度层需要真实工具链配合，至少要把一条视频链和一条文本链稳定起来

### 第三优先级

- `网页 UI`

原因：

- 现在 UI 已经有工作台壳子，短期内重点是继续贴合真实能力，不是纯视觉重做

## 5. 每条主线的完成标准

### UI 完成标准

- 6 步工作台结构稳定
- 每个步骤都有清晰空状态
- 真实任务状态和 provider 状态能看懂

### API 完成标准

- 至少 1 条文本链可用
- 至少 1 条视频链可用
- 至少 1 条语音链可用或有降级方案
- 每条链都有明确报错

### 导演 Agent 完成标准

- 能先生成调度计划，再驱动后续执行
- 能记录当前选择了什么模型
- 能在失败时自动切 fallback
- job 里能看到完整执行轨迹

## 6. 接下来怎么推进

短期内按这 3 条线来：

1. `导演 Agent`：先补总调度层
2. `API`：把 Vidu / 混元 / 兼容聚合链接稳
3. `UI`：把调度结果、模型选择和失败原因展示出来

一句话：

这个项目从现在开始，不再按“一个功能一个功能”推进，而是按 `UI / API / 导演 Agent` 三条主线推进。
