# AI 导演网页规范化开发计划：今晚最终汇报草稿

更新时间：2026-04-28

## 1. 一句话结论

今天没有继续盲目堆功能，而是把 AI 导演网页的开发方式改成“规格先行、任务拆分、状态验收、截图 QA”的稳定流程。后续每个 Agent 都能按文档接力，不需要重新猜页面应该怎么做。

## 2. 今天建立的规范体系

已落地 9 份文档：

1. `AI导演工作台页面规格.md`：定义产品目标、第一屏、页面区域、响应式规则、明确不要什么。
2. `AI导演网页规范化推进计划.md`：把 GitHub/开源方法转成当前项目开发流程。
3. `AI导演网页Agent执行规范.md`：规定每个 Agent 开发前读什么、能改什么、怎么验收。
4. `AI导演网页验收矩阵.md`：固定视口、固定状态、任务验收表、截图 QA 模板。
5. `AI导演竞品观察-小云雀.md`：提炼小云雀的能力入口、创意公式、多候选、排队解释机制。
6. `AI导演任务1-2实施拆分.md`：把首屏总览和剧本可视化拆成 10 个可执行小任务。
7. `AI导演当前网页问题诊断.md`：按 P0/P1/P2 梳理当前网页体验问题。
8. `AI导演今晚交付大纲.md`：固定今晚最终交付结构。
9. `AI导演任务1.1实施提示词.md`：为第一个实际页面小任务准备可直接执行的提示词。

## 3. 学到的开源/GitHub 方法

| 工具/方法 | 学到什么 | 当前怎么用 |
| --- | --- | --- |
| awesome-cursorrules | Agent 需要项目级规则，不能每次临时解释 | 已写 Agent 执行规范 |
| Playwright | 网页要用真实浏览器、多视口、真实交互验收 | 暂不安装，先用 in-app browser 人工截图 QA |
| Storybook | 每个 UI 模块都要覆盖 empty/loading/error/completed 状态 | 已转成状态验收矩阵 |
| axe-core | 按钮、表单、状态文案要可访问，不只靠颜色 | 已写入 UI 验收底线 |
| Lighthouse | 重图片/视频会拖垮体验，要限制素材加载 | 已写入性能风险检查 |

暂不安装这些工具。原因是当前阶段用户要完整规划，不做工具链重构；安装第三方工具以后单独确认。

## 4. 从竞品学到的机制

### 小云雀

可借鉴：

- 入口按能力分类，而不是一个按钮包打天下。
- 展示 AI 思考过程/创意公式。
- 一次生成多个候选，用户选一个。
- 支持自有素材和 AI 素材融合。
- 排队生成要解释清楚。

转成当前网页：

- 增加“导演公式卡”。
- 总览显示“已选用/待选用”。
- LibTV 等待和超时文案改成排队/稍后同步，而不是失败感。

### TapNow / LTX 类 storyboard

可借鉴：

- 剧本拆成可编辑 storyboard。
- 每个镜头是可验收卡片。

转成当前网页：

- 文字分镜显示对应视频素材状态。
- 明确区分“文字分镜”和“视频镜头”。

### Runway / Vidu 类参考素材

可借鉴：

- 参考图和候选版本影响后续生成。

转成当前网页：

- 选用素材进入剪辑面板和成片上下文。
- 后续补参考素材上传入口。

### 达芬奇式剪辑面板

可借鉴：

- 素材池、预览、检查器、时间线要在同一工作台中。

转成当前网页：

- 保留现有剪辑面板结构，后续做截图 QA 和移动端打磨。

## 5. 当前网页最大的 5 个问题

1. 无任务空状态太弱：用户不知道输入后会生成什么。
2. Provider 状态容易误导：备用能力 error 会让人以为主链坏了。
3. 缺少导演公式中间层：用户看不到 AI 如何拆解想法。
4. 结果总览没有充分表达“还缺什么”：下一步动作不够明确。
5. 文字分镜和视频镜头关系不够清楚：用户可能误以为有文字分镜就已有视频。

## 6. 任务 0-7 拆分

| 任务 | 目标 | 下一步 |
| --- | --- | --- |
| 任务 0：Agent 执行规范 | 让后续 Agent 按统一规则工作 | 已完成文档，后续执行时强制先读 |
| 任务 1：首屏和结果总览 | 第一屏解释清楚结果和下一步 | 做空状态、Provider 降噪、总览升级 |
| 任务 2：剧本可视化 | 剧本变成可验收导演文档 | 加导演公式卡、分镜状态、局部调整入口 |
| 任务 3：角色/场景/道具主体图 | 主体图能生成、展示、选用 | 总览显示选用状态 |
| 任务 4：分镜到视频镜头 | 分镜按顺序转视频 | 明确文字分镜/视频镜头区别 |
| 任务 5：剪辑面板 | 表达素材如何进入成片 | 做截图 QA 和移动端打磨 |
| 任务 6：最终成片和同步 | 成片等待和同步可理解 | 优化排队/超时文案 |
| 任务 7：截图 QA | 固化桌面/手机验收 | 做首轮 QA 记录 |

## 7. 首批 5 个实际执行任务

推荐顺序：

1. 任务 1.1：空状态首屏说明。
2. 任务 1.2：Provider 状态降噪。
3. 任务 1.3：结果总览状态升级。
4. 任务 2.2：导演公式卡。
5. 任务 2.4：文字分镜验收状态。

为什么先做 1.1：

- 风险最低。
- 不碰后端。
- 不调用 GPT。
- 不调用 LibTV。
- 但能立刻改善用户第一眼“不知道结果在哪里”的问题。

## 8. 截图 QA 计划

固定视口：

- 1440 x 900 桌面。
- 390 x 844 手机。

固定状态：

- 无任务。
- 生成中。
- 有剧本无素材。
- 有素材未成片。
- 已成片。
- 失败/超时。
- 长文本。

检查项：

- 第一屏是否解释清楚。
- 结果是否有入口。
- 按钮是否溢出。
- 卡片是否重叠。
- Provider 是否误导。
- 图片/视频是否撑坏页面。
