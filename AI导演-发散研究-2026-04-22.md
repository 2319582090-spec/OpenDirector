# AI 导演项目发散研究

日期：2026-04-22  
用途：给明天继续聊产品方向、技术路线、差异化定位时直接参考。

## 1. 先说我现在对项目的判断

你这个项目其实有三条不同的路，而且它们不是一回事：

1. `导演工作台`
   - 关键词：从想法到样片、可控、半自动、导演视角、交付效率。
   - 对标更像 [LTX Studio](https://ltx.studio/platform/ai-movie-maker)、[Creatorwood](https://www.creatorwood.tv/)、[Story67](https://story67.ai/)。
   - 这条路最适合你现在做，因为它最容易在 1 个月内做出一个真实能演示、能收费试单的东西。

2. `AI 电影工厂`
   - 关键词：批量生产、模板化、角色一致性、客户交付、商业片和短剧流水线。
   - 对标更像 Creatorwood 的“Movie Machine”+ 代理/工作流平台。
   - 这条路更偏 B 端和内容工厂。

3. `互动式无限叙事平台`
   - 关键词：观众输入、剧情分叉、世界观持续演化、无限剧情、可玩电影。
   - 对标更像 [Roto AI / Roto TV](https://rotopus.ai/)、[Showrunner](https://www.showrunner.xyz/)、[Story AI](https://storyai.cc/)。
   - 这条路想象力最大，但比“导演工作台”难至少一个量级，因为难点从“生成一条片”变成了“维护一个世界和无限分支”。

我的判断是：

- `短期应该做 1`
- `中期可以吸收 2 的交付能力`
- `长期再去碰 3 的无限剧情`

也就是说，你现在最危险的事不是做慢，而是把 `工作台产品`、`内容工厂`、`互动叙事平台` 三个东西混在一起，导致第一版做不出来。

## 2. Roto AI 给你的启发

我看了 [Roto AI / Roto TV 官网](https://rotopus.ai/)。它的核心不是“帮你生成一个电影”，而是：

- 让用户输入一句策略、一个道具、一句对白
- 系统立刻生成新的剧情路径
- 本质上是 `观众输入 -> 故事状态变化 -> 生成下一段内容`

它强调的是：

- `interactive open-world video`
- `branching cinematic content`
- `creator partner / waitlist`
- “不是看故事，而是参与故事”

这说明一件很重要的事：

`Roto 的护城河不是视频模型本身，而是互动叙事引擎。`

也就是：

- 世界状态怎么记忆
- 人物关系怎么保持
- 用户输入如何影响剧情
- 分叉后如何不崩
- 如何实时生成“下一段”而不是一次性整片

所以如果你未来想做“无限剧情、无限创意”的电影平台，重点不是先去找最强视频模型，而是要先定义：

1. 世界观状态模型
2. 角色记忆与关系图谱
3. 剧情推进规则
4. 用户输入对剧情的影响边界
5. 低延迟的内容生成与缓存机制

换句话说：

`Roto AI 更像“AI 互动影视引擎”，而你现在更适合先做“AI 导演工作台”。`

## 3. 同类产品我会怎么分组看

### A. 从脚本到样片的一站式工作台

- [LTX Studio](https://ltx.studio/platform/ai-movie-maker)
  - 官方直接写的是 “script to screen”
  - 重点是故事板、镜头控制、场景生成、导出 pitch deck
  - 我觉得它最值得你学习的是：`它不是纯聊天，而是导演流程界面`

- [Creatorwood](https://www.creatorwood.tv/)
  - 它更进一步，想把“写作 -> 电影 -> 分发 -> 收费”全打通
  - 它的产品想象不是单次生成，而是创作者经济
  - 它很像“作者把文字 IP 直接变片，再卖给观众”

- [Story67](https://story67.ai/)
  - 更偏“快速生成成片”的消费级 / 轻商用产品
  - 它把模式拆得很清楚：Story Mode、Talking Video、Series Mode、News 等
  - 这说明产品化时，一个很重要的能力不是“更强模型”，而是 `把不同生成意图包装成模式`

### B. 互动式 AI 叙事 / 可玩影视

- [Roto AI / Roto TV](https://rotopus.ai/)
  - 重点是观众驱动、开放世界、分支叙事
  - 更像“AI 可玩电影”

- [Showrunner](https://www.showrunner.xyz/)
  - 官方首页已经在讲 “take the director’s seat”
  - 它不是传统视频工具，而是 `生成动画节目和角色世界`
  - 这个方向更接近“AI 电视台 / AI 世界观”

- [Story AI](https://storyai.cc/)
  - 更轻，先从“一个 premise -> 一段可玩的 opening scene”开始
  - 它启发很大：`不一定一上来就做完整电影，可以先做“可玩的开场 + 第一分支”`

## 4. 你项目的真正差异化，可能不在模型而在“导演语法”

大厂和平台都能接模型，但不是每个产品都懂“导演怎么思考”。

你如果要跟大厂错位，我觉得最值得打的不是：

- 我也有视频生成
- 我也能配音
- 我也能自动剪辑

而是：

- 我更懂 `导演决策`
- 我更懂 `提案样片`
- 我更懂 `镜头语言`
- 我更懂 `从想法到可展示样片` 这个真实流程

所以你的产品可以不是“万能视频生成平台”，而是：

`导演型 AI 创作台`

更具体一点，差异化可以做在：

1. `导演模式`
   - 悬疑预告片
   - 品牌概念片
   - AI 漫剧试片
   - 电影提案样片
   - 角色测试片

2. `导演控制点`
   - 开场情绪
   - 角色登场方式
   - 镜头密度
   - 节奏曲线
   - 旁白气质
   - 尾镜头记忆点

3. `导演介入点`
   - 用户不是改 prompt，而是改“镜头意图”
   - 例如：把第 3 镜头压短、让角色更孤独、尾镜头更像品牌片、把冲突提前一镜头

如果做到这层，产品就会从“AI 工具集成页”变成“导演工作流系统”。

## 5. 上下游最值得接的模型与软件

我按“最适合你现在做网页 MVP”的视角来排，不是按论文最强排。

### 上游：创意理解 / 剧本 / 分镜

- `OpenRouter / GPT-5.4`
  - 适合做提案文案、剧情梳理、镜头语言、导演建议
  - 优点：文字审美和结构能力稳定

- `DashScope / Qwen`
  - 更适合中国内地接入和成本控制
  - 适合做结构化 JSON 输出、批量任务

- `Gemini`
  - 适合多模态理解和一些图文结合任务
  - 但在你现在这条 MVP 线上，不一定是第一优先级

我的建议：

- `脚本与镜头规划层：OpenRouter + Qwen 双轨`
- OpenRouter 出“更像提案”的版本
- Qwen 出“更像结构化生产”的版本

### 中游：视频生成

- [阿里百炼 Wan 视频](https://help.aliyun.com/zh/model-studio/text-to-video-guide)
  - `wan2.7-t2v` / `wan2.6-t2v` 已支持多镜头叙事、声画同步
  - 官方文档显示可做 2 到 15 秒、30fps、MP4
  - 这对你做“多镜头 Web MVP”非常关键

- [Runway Gen-4.5](https://runwayml.com/pricing)
  - 更偏高质量、国际化、创意行业认知强
  - 官方价格页显示：`625 credits = 25s of Gen-4.5`
  - 更适合做高价值镜头、备用链、品牌感样片

- [Luma Ray3](https://lumalabs.ai/ray)
  - 更适合强调一致性、控制、keyframe、video-to-video
  - 更像你后面应该评估的“增强链路”

我的建议：

- `主链：Wan`
- `备用：Runway`
- `探索增强：Luma`

### 声音层

- [ElevenLabs TTS](https://elevenlabs.io/docs/overview/capabilities/text-to-speech)
  - 适合高质量旁白、情绪感和商业视频配音

- [Qwen-TTS](https://help.aliyun.com/zh/model-studio/qwen-tts)
  - 更适合中国内地、成本更友好、接入统一

- [Suno](https://suno.com/l/make-music-with-ai) / [Udio](https://help.udio.com/en/articles/10716509-explore-the-udio-interface)
  - 更适合 BGM 探索
  - 第一版其实可以不用“真正的 AI 作曲”，可以先用免版权音乐或模板 BGM

我的建议：

- `旁白：ElevenLabs 主链 + Qwen-TTS 备用`
- `音乐：先不把 AI 音乐做成第一阶段关键路径`

## 6. GitHub 上我觉得值得你看的自动剪辑 / 时间线 / 字幕栈

这里我不是随便列 repo，而是按“你这个产品真能用上”的角度排。

### 第一层：你现在最适合直接吸收的

- [Remotion](https://github.com/remotion-dev/remotion)
  - 用 React 做视频，特别适合你这种 Web 产品
  - 不是传统 NLE，但特别适合模板化、程序化导出、自动字幕、品牌包装

- [auto-editor](https://github.com/WyattBlue/auto-editor)
  - 它的定位非常明确：effort free video editing
  - 对播客、口播、采访、停顿删除很有帮助
  - 如果你后面做“自动粗剪”或“去废话/去停顿”的链路，它很值得吸收

- [WhisperX](https://github.com/m-bain/whisperx)
  - 自动转写 + 单词级时间戳 + diarization
  - 这是你做高级字幕、对白切镜、口播节奏识别的关键基础设施

- [PySceneDetect](https://github.com/Breakthrough/PySceneDetect)
  - 自动切镜头、做镜头边界检测
  - 很适合拿来做“镜头重组”“素材粗分段”

- [OpenTimelineIO](https://github.com/AcademySoftwareFoundation/OpenTimelineIO)
  - 这是一个非常重要但容易被忽略的东西
  - 如果你后面想从“自动生成一条片”走向“可编辑时间线”，OTIO 会是很好的中间表示层

### 第二层：值得参考但不一定现在就引入

- [OpenCut](https://github.com/OpenCut-app/OpenCut)
  - 开源 CapCut 替代方向
  - 值得看的是它的 web video editor 形态，而不是直接拿来嵌进你的 MVP

- [Vanta](https://github.com/itsjwill/vanta)
  - 一个基于 Remotion 的开源 AI 视频引擎
  - 更像“视频创作操作系统”的方向
  - 适合你研究它的时间线、字幕、动效、模板架构

- [VectCutAPI](https://github.com/sun-guannan/VectCutAPI)
  - 这个方向挺值得看，因为它明确在做 `toAgent` 的视频编辑 API
  - 对你未来“让 agent 学会剪辑接口”很有启发

- [vidpipe](https://github.com/htekdev/vidpipe)
  - 很像“自动把录屏/长视频处理成社媒成片”的 agentic pipeline
  - 对你理解“自动后处理”有帮助

## 7. 我最看好的技术组合，不是一个模型，而是一条链

如果让我按现在的可落地性给你配一条最现实的链：

### 版本 A：最快闭环

- 创意 / 剧本：OpenRouter GPT-5.4
- 结构化镜头：Qwen
- 旁白：ElevenLabs
- 视频：Wan
- 剪辑：Remotion + FFmpeg
- 字幕：WhisperX 或直接脚本生成字幕

### 版本 B：更适合国内长期成本控制

- 创意 / 镜头：Qwen
- 旁白：Qwen-TTS
- 视频：Wan
- 剪辑：Remotion + FFmpeg
- 时间线：后面接 OTIO

### 版本 C：高质量样片链

- 创意：OpenRouter GPT-5.4
- 镜头规划：Qwen
- 视频主镜头：Runway
- 补镜 / 稳定输出：Wan
- 旁白：ElevenLabs
- 包装导出：Remotion + FFmpeg

## 8. 如果你以后真的想做“无限剧情电影”，技术上会多出什么

如果真的走 Roto / Showrunner 方向，会多出一整层系统：

1. `Story State Engine`
   - 角色关系、物品、地点、秘密、目标、冲突都要可追踪

2. `Branch Memory`
   - 每次用户输入不是一次性 prompt，而是对世界状态的修改

3. `Narrative Rules`
   - 哪些剧情能发生，哪些不能发生
   - 角色不能前后人格崩掉

4. `Episode Compiler`
   - 把新的剧情节点编译成镜头、对白、旁白、过场

5. `Low-latency Cache`
   - 常见分支、角色镜头、场景资产需要缓存，不然交互等待会太长

所以我会建议：

`先把“导演工作台”做成，再把“无限剧情引擎”当作第二产品。`

## 9. 我现在最想帮你验证的 5 个问题

1. 你的用户最先愿意为哪一种结果付钱？
   - 提案样片
   - AI 漫剧试片
   - 品牌概念片
   - 招商 / 路演视频

2. 你是更想做 `导演工具`，还是 `自动出片工厂`？

3. 你的第一批用户是：
   - 独立创作者
   - AI 漫剧团队
   - 小型内容工作室
   - 想把文字 IP 变成视频的人

4. 你最先做的是：
   - 固定模板高成功率
   - 自由创作高想象力

5. 你未来到底要不要做“观众参与剧情”？
   - 如果要，那是第二阶段的世界观引擎，不应该混在第一版里

## 10. 我现在给你的一个直觉结论

如果只从“最有机会做出来”的角度看，我觉得你的项目最强的一句定位可能不是：

- 人人都能做电影

而是：

`把导演工作流做成一个网页，让一个想法最快速地变成可展示、可提案、可收费的样片。`

这句话更落地，也更容易做出产品。

如果以后再往前推一步，再升级成：

`导演工作台 -> 内容工厂 -> 互动式世界叙事平台`

这个顺序会比一上来就想做“无限剧情电影宇宙”稳很多。

## 11. 明天我建议我们接着聊的顺序

1. 先定第一版的唯一用户是谁
2. 再定第一版只做哪一种片
3. 再定网页上必须出现哪 3 个导演控制点
4. 再决定第二阶段要不要加“分支剧情 / 无限叙事”
